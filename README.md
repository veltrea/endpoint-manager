# Endpoint Manager

ローカルネットワーク上の LM Studio インスタンスを自動検出して、エンドポイント情報を一元管理するサービスです。

## なぜ作ったか

VS Code をフォークして、ローカル LLM に特化した AI IDE（Cursor や Google Antigravity のようなもの）を開発しています。ただ、バイブコーディングで大量のコードを生成していると、コードベースが AI の扱い切れる行数を超えたのか、破綻することが繰り返されるようになりました。

そこで、IDE の中に抱え込まなくてもいい機能は外に出してしまうことにしました。エンドポイント管理もそのひとつです。外に出したついでに MCP サーバー化して、単体でも使えるようにしてあります。

## 何ができるか

- **LM Studio の自動検出** — 同一ネットワークセグメント内にある複数デバイス上の LM Studio を自動でスキャンして見つけます
- **モデル情報の管理** — ダウンロード済みの AI モデルを一覧化して、どのマシンで何が使えるかを把握できます
- **ロールベースのルーティング** — 「コーディング用」「高速応答用」のようにロールを割り当てて、用途に応じたエンドポイント解決ができます
- **ヘルスチェック** — オンライン/オフラインの監視と自動復帰チェック

## 事前準備

### 必須：LM Studio のサーバー機能を有効にする

LM Studio はデフォルトではサーバーとして動いていません。サーバーが起動していないと、同一マシンであっても Endpoint Manager からは検出できません。

GUI の「Developer」タブからサーバーを起動するか、ヘッドレスモード（`lms server start`）で起動してください。

### ネットワーク越しに使う場合は追加設定が必要

別のマシンにある LM Studio を検出するには、さらに以下が必要です：

- **LM Studio の「Serve on Local Network」を有効にする** — デフォルトでは localhost のみにバインドされるため、ネットワーク上の他のマシンからアクセスできません
- **ファイアウォールでポート 1234（LM Studio のデフォルト）を開放する** — macOS はシステム設定 → ネットワーク → ファイアウォールから、Windows は Windows Defender ファイアウォールの受信規則から設定できます

## MCP サーバーとして使う

MCP をインストールすれば、Claude Code や Gemini CLI など MCP 対応の AI エージェントから、ローカルのエンドポイント情報を自動的に取得できるようになります。

```json
{
  "mcpServers": {
    "endpoint-manager": {
      "command": "npx",
      "args": ["tsx", "mcp/server.ts"],
      "cwd": "/path/to/endpoint-manager"
    }
  }
}
```

提供されるツール：

| ツール | 説明 |
|---|---|
| `list_endpoints` | 検出済みの全エンドポイントを一覧表示 |
| `resolve_endpoint` | ロールやモデル指定で最適なエンドポイントを解決 |
| `list_roles` | ロール割り当ての一覧 |
| `scan_network` | ネットワーク再スキャンを実行 |
| `assign_role` | エンドポイントにロールを割り当て |
| `chat_completion` | エンドポイントを自動解決して推論を実行 |

```
┌──────────────────────────────────┐
│  Tauri Desktop App (src-tauri/)  │
│  Rust の殻。バックエンドを管理。   │
└──────────┬───────────────────────┘
           │ spawn
┌──────────▼───────────────────────┐
│  Node.js Backend (src/)          │
│  WS :3797 + HTTP :3798           │
│  スキャン・ヘルス監視・モデル管理   │
└──────────▲───────────────────────┘
           │ ws://127.0.0.1:3797
┌──────────┴───────────────────────┐
│  MCP Server (mcp/)               │
│  stdio transport                 │
│  AI エージェントから利用           │
└──────────────────────────────────┘
```

## 起動方法

```bash
# バックエンドのみ（ヘッドレス）
npm start

# Tauri デスクトップアプリ
npm run tauri:dev

# MCP サーバー経由（バックエンド未起動なら自動で立ち上がります）
cd mcp && npx tsx server.ts
```

## テスト

```bash
npm test              # 全テスト（43テスト）
npm run test:unit     # ユニットテストのみ
npm run test:integration  # WS プロトコル統合テスト
npm run test:e2e      # MCP ツール呼び出し e2e テスト
```

## こんな使い方が面白いかも

Mac を複数台持っている人なら、それぞれに LM Studio を入れて小型モデルを分散配置し、Endpoint Manager で一括管理するという使い方ができます。用途ごとにロールを振り分ければ、複数モデルへの並行リクエストも視野に入ります。

## 開発状況

このプロジェクトは開発途中です。エンドポイントの自動検出やモデル一覧の取得など基本的な部分は動いていますが、まだ作り込みが足りない機能もあります。

- **ロールベースのルーティング** — 仕組みはあるけど実装が中途半端で、現状はまだ役に立ちません。今のところは「ローカルのエンドポイントをサクッと一覧で取れる」くらいのつもりで使ってもらえると
- **複数エンドポイントへの並行リクエスト** — 構想段階
- **Tauri デスクトップアプリ** — 基本動作はするけど、トレイ常駐やビルド周りの整備がまだ
- **テスト** — ユニット〜e2e まで 43 テスト通ってますが、カバレッジはまだまだ

追い追い追加していきます。
